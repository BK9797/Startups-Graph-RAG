# CYPHER.md — every Cypher query this project can run

This is the **complete list**. There is no LLM in the loop when it comes
to Cypher: a natural-language question is matched, by regex, against one
of the templates below (implemented in `app/core/cypher_library.py`),
and *that* fixed, parameterized query — never anything generated by a
model — is what runs against Neo4j.

If a question matches nothing here, the app runs exactly one other
query: the fulltext fallback search at the bottom of this file. If that
also finds nothing, the API returns an "unanswerable" response and never
touches Neo4j again for that request.

The only LLM call anywhere in the pipeline happens *after* one of these
queries has already run — turning the returned rows into a
natural-language sentence. It cannot write, choose, or alter Cypher.

How to read each entry:
- **Matches** — the phrasing patterns that route to this template (not exhaustive, see the actual regexes in `cypher_library.py` for the precise grammar)
- **Params** — what gets extracted from the question and bound as Neo4j query parameters (never string-interpolated into the query)
- **Cypher** — the exact, fixed query text

## 1. `founders_of_company`
The founder(s) of a given company, and the founding year.

**Matches:** "Who founded X?", "Who is/are the founder(s) of X?", "Who started X?"
**Params:** `name` (company name fragment)

```cypher
MATCH (p:Person)-[r:FOUNDED]->(c:Company)
WHERE toLower(c.name) CONTAINS toLower($name)
RETURN p.name AS founder, c.name AS company, r.year AS year
ORDER BY r.year LIMIT $limit
```

## 2. `serial_founders`
Founders who have founded more than one company. No entity in the question.

**Matches:** "serial founders", "who has founded more than one company", "founders of multiple companies"
**Params:** none

```cypher
MATCH (p:Person)-[:FOUNDED]->(c:Company)
WITH p, count(DISTINCT c) AS num, collect(DISTINCT c.name) AS companies
WHERE num > 1
RETURN p.name AS founder, num AS companies_founded, companies
ORDER BY num DESC LIMIT $limit
```

## 3. `person_profile`
A person's full profile: companies founded, work history, awards won.

**Matches:** "What has X founded?", "Who is X?"
**Params:** `name` (person name fragment)

```cypher
MATCH (p:Person) WHERE toLower(p.name) CONTAINS toLower($name)
OPTIONAL MATCH (p)-[:FOUNDED]->(fc:Company)
OPTIONAL MATCH (p)-[w:WORKS_AT]->(wc:Company)
OPTIONAL MATCH (p)-[:WON]->(a:Award)
RETURN p.name AS person, p.born AS born, p.hometown AS hometown,
       collect(DISTINCT fc.name) AS founded_companies,
       collect(DISTINCT {company: wc.name, role: w.role, current: w.current}) AS roles,
       collect(DISTINCT a.name) AS awards LIMIT $limit
```

## 4. `company_profile`
A company's full profile: founders, investors, valuation, description.

**Matches:** "Tell me about X", "What is X?"
**Params:** `name` (company name fragment)

```cypher
MATCH (c:Company) WHERE toLower(c.name) CONTAINS toLower($name)
OPTIONAL MATCH (p:Person)-[:FOUNDED]->(c)
OPTIONAL MATCH (i:Investor)-[:INVESTED_IN]->(c)
RETURN c.name AS company, c.sector AS sector, c.founded AS founded,
       c.headquarters AS headquarters, c.valuation_billion AS valuation_billion,
       c.description AS description, collect(DISTINCT p.name) AS founders,
       collect(DISTINCT i.name) AS investors LIMIT $limit
```

## 5. `current_team_at_company`
People who **currently** work at a given company.

**Matches:** "current executives/team/leadership at X"
**Params:** `name` (company name fragment)

```cypher
MATCH (p:Person)-[r:WORKS_AT {current: true}]->(c:Company)
WHERE toLower(c.name) CONTAINS toLower($name)
RETURN p.name AS person, r.role AS role LIMIT $limit
```

## 6. `all_employees_of_company`
Everyone who has **ever** worked at a company (past and present).

**Matches:** "who works/worked at X", "employees of X"
**Params:** `name` (company name fragment)

```cypher
MATCH (p:Person)-[r:WORKS_AT]->(c:Company)
WHERE toLower(c.name) CONTAINS toLower($name)
```

## 7. `investor_portfolio`
Companies a given investor has invested in.

**Matches:** "what companies has X invested in", "X's portfolio", "portfolio of X"
**Params:** `name` (investor name fragment)

```cypher
MATCH (i:Investor)-[r:INVESTED_IN]->(c:Company)
```


## 9. `investors_by_type`
Investors filtered by investor type.

**Matches:** "which investors are venture capital / angel network / private equity / growth equity / seed fund / accelerator"
**Params:** `type` (one of the six type strings above)

```cypher
MATCH (i:Investor) WHERE toLower(i.type) CONTAINS toLower($type)
RETURN i.name AS investor, i.hq AS hq, i.founded AS founded LIMIT $limit
```

## 10. `companies_by_sector_and_valuation`
Companies filtered by sector (optional) and a valuation threshold.

**Matches:** "which \<sector\> companies are valued over/under/at least/... $N billion", "companies valued over $N billion" (sector omitted)
**Params:** `sector` (optional, may be `null`), `amount` (float, in $B)
**Note:** the comparison operator (`>`, `<`, `>=`, `<=`, `=`) is chosen from a fixed 10-entry whitelist mapping words like "over"/"under"/"at least" to symbols, and is the one piece of this query built into the query text itself rather than passed as a parameter — never from unfiltered user text.

```cypher
-- with a sector:
MATCH (c:Company) WHERE toLower(c.sector) CONTAINS toLower($sector)
  AND c.valuation_billion {op} $amount
RETURN c.name AS company, c.sector AS sector, c.valuation_billion AS valuation_billion
ORDER BY c.valuation_billion DESC LIMIT $limit

-- without a sector:
MATCH (c:Company) WHERE c.valuation_billion {op} $amount
RETURN c.name AS company, c.sector AS sector, c.valuation_billion AS valuation_billion
ORDER BY c.valuation_billion DESC LIMIT $limit
```

## 11. `top_valuation_companies`
Companies ranked by valuation, highest first. No entity in the question.

**Matches:** "most valuable companies", "top companies by valuation", "highest-valued startups"
**Params:** none

```cypher
MATCH (c:Company) WHERE c.valuation_billion IS NOT NULL
RETURN c.name AS company, c.sector AS sector, c.valuation_billion AS valuation_billion
ORDER BY c.valuation_billion DESC LIMIT $limit
```

## 13. `company_products`
Products developed by a given company.

**Matches:** "what products does X develop/make/build/offer", "products by/from/of X"
**Params:** `name` (company name fragment)

```cypher
MATCH (c:Company)-[:DEVELOPS]->(pr:Product)
```

## 14. `entity_awards`
Awards won by a given person **or** company (polymorphic on node label).

**Matches:** "what awards has X won", "awards won by X", "awards for X"
**Params:** `name` (person or company name fragment)

```cypher
MATCH (n)-[:WON]->(a:Award)
WHERE (n:Person OR n:Company) AND toLower(n.name) CONTAINS toLower($name)
RETURN n.name AS entity, labels(n)[0] AS entity_type,
       a.name AS award, a.category AS category, a.year AS year
ORDER BY a.year LIMIT $limit
```

## 15. `who_acquired_company`
Which company acquired a given target.

**Matches:** "who acquired X", "which company acquired X"
**Params:** `name` (target company name fragment)

```cypher
MATCH (acq:Company)-[r:ACQUIRED]->(tgt:Company)
WHERE toLower(tgt.name) CONTAINS toLower($name)
RETURN acq.name AS acquirer, tgt.name AS target,
       r.year AS year, r.amount_million AS amount_million
ORDER BY r.year LIMIT $limit
```

## 16. `acquisitions_by_company`
Companies acquired by a given acquirer (inverse of #15).

**Matches:** "what did X acquire", "which companies did X acquire", "what has X acquired"
**Params:** `name` (acquirer company name fragment)

```cypher
MATCH (acq:Company)-[r:ACQUIRED]->(tgt:Company)
WHERE toLower(acq.name) CONTAINS toLower($name)
RETURN tgt.name AS target, r.year AS year, r.amount_million AS amount_million
ORDER BY r.year LIMIT $limit
```

## 17. `top_acquirers`
Companies ranked by number of acquisitions made. No entity in the question.

**Matches:** "which company made the most acquisitions", "top acquirers", "who has acquired the most companies"
**Params:** none

```cypher
MATCH (acq:Company)-[:ACQUIRED]->(tgt:Company)
RETURN acq.name AS acquirer, count(tgt) AS acquisitions
ORDER BY acquisitions DESC LIMIT $limit
```

## 18. `list_sectors`
Distinct sectors present in the graph. No entity in the question.

**Matches:** "what sectors exist/are there/are covered", "list all sectors", "which sectors"
**Params:** none

```cypher
MATCH (c:Company) WHERE c.sector IS NOT NULL
RETURN DISTINCT c.sector AS sector ORDER BY sector LIMIT $limit
```

## Fallback: `fulltext_fallback`
Used when (a) no template above matches the question at all, or
(b) a template matched but its query returned zero rows (e.g. the name
in the question doesn't exactly match what's stored). Requires a Neo4j
fulltext index named `entity_search` over node `name`/`description`
properties (created by `scripts/load_neo4j.py`).

**Params:** `term` — capitalized word sequences pulled from the question, with a trailing `~` for fuzzy matching; falls back to the raw question text if the question has no capitalized words.

```cypher
CALL db.index.fulltext.queryNodes('entity_search', $term) YIELD node, score
RETURN labels(node)[0] AS label, node.name AS name, score
ORDER BY score DESC LIMIT 10
```

## Safety notes

- Every query above always includes both a `RETURN` clause and a `LIMIT`
  clause (`LIMIT $limit`, capped server-side by `MAX_CYPHER_ROWS`).
- `graph_rag.validate_cypher()` re-checks both of those, plus a
  blocklist of write/admin keywords (`CREATE`, `MERGE`, `SET`, `DELETE`,
  `DETACH`, `REMOVE`, `DROP`, `LOAD CSV`, `apoc.periodic.*`,
  `apoc.create.*`, `apoc.merge.*`, `GRANT`, `REVOKE`, `DENY`), on every
  resolved query before it runs — belt-and-suspenders, since none of the
  fixed templates could produce one of those anyway.
- `Neo4jClient.run_read()` additionally executes every query inside an
  explicit Neo4j **read** transaction, so even a hypothetical bug in a
  future template couldn't write to the database — the driver would
  reject it.
- All parameters (`$name`, `$sector`, `$amount`, `$year`, `$type`,
  `$term`, `$limit`) are bound through the Neo4j driver's parameter
  mechanism, never string-formatted into the query. The one exception —
  the comparison operator in template #10 — comes from a fixed 10-entry
  whitelist (`_OPERATORS` in `cypher_library.py`), not from arbitrary
  user text.

## Keeping this file in sync

If you add, remove, or change a template in `app/core/cypher_library.py`,
update the corresponding entry here in the same change. `tests/test_cypher_library.py`
asserts every template's query passes `validate_cypher()`, but it doesn't
check this file for drift — that's on the humans (or the LLM) making the change.
